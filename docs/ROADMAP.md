# LeakWatch 로드맵 (MVP → V1)

문서 기준일: 2026-02-11 (KST)

## 목표 정의
- MVP(2주): “업로드 → AI 정규화 → 누수 5종 탐지 → 근거+절감액 → 이메일 초안/증빙 자동 생성 → 승인 후 발송”까지 실제로 동작
- V1(6~8주): 이메일 포워딩, 설치 앱 동기화 안정화, 에이전시 포털, 리포트 자동화 완성, 정확도/비용 최적화

## MVP(2주) 범위
1) Shopify 스토어 연결(임베디드 앱) + 멀티테넌시(Org/Shop)
2) 인보이스 업로드(PDF + CSV + 이미지 중 최소 2종) + 저장 + 버전 관리
3) LLM 표준화(JSON) + 스키마 검증/재시도
4) 누수 탐지 v1: 최소 5가지 유형 + 근거(원문 텍스트 스팬/행/페이지)
5) 절감 기회 대시보드(Top leaks, 예상 절감액, 신뢰도)
6) 액션 센터:
   - 이메일 초안 자동 생성
   - 증빙 패키지(Zip/PDF) 생성
   - 승인 플로우
   - 발송/기본 추적(Delivered/Failed)
7) 주간/월간 리포트(앱 내 + 이메일 발송)
8) 기본 보안: 토큰 암호화, RBAC, 감사로그 최소

## MVP 비범위(명시)
- 벤더 계정에 로그인해서 “실제 해지 버튼 클릭”까지 자동 실행 (불가/리스크 큼)
- 카드/은행/Stripe/PayPal 실시간 커넥터(초기에는 업로드/포워딩으로 대체)
- 매출/광고비/COGS 포함 Profit analytics(LeakWatch는 “누수”에 집중)

## V1(6~8주) 확장
- 이메일 포워딩 정식 지원(인바운드 파싱, 자동 문서 생성)
- Shopify 설치 앱 목록 동기화 안정화 + “설치 안 됐는데 청구됨” 탐지 강화
- 벤더 연락처(지원 이메일/환불 정책 링크) 자동 수집/캐싱 (가능 범위)
- 에이전시 멀티 스토어 포털 + 클라이언트별 Top5 누수 리포트 자동 생성
- 탐지 정확도 개선(벤더 canonicalization, 중복결제/트라이얼 탐지 고도화)
- LLM 비용 최적화(캐시/배치/요약, low-cost 모델 우선)
- 감사로그/보안 강화, 데이터 삭제 자동화(GDPR)

## 성공 지표(로드맵 게이트)
- Activation: 연결 후 24시간 내 “첫 리포트 생성” 40%+
- Time-to-Value: 첫 세션 30분 내 “절감 후보 1개 이상” 60%+
- Action CTR: LeakFinding 상세에서 액션 생성률 25%+
- Approve Rate: 생성된 액션의 승인/발송률 30%+
- Retention(4주): 주간 리포트 on 상태 유지율 70%+
- ROI: 고객 당 월 절감액 / 월 구독료 >= 5배 (중앙값 기준)
